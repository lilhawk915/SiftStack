SiftStack Daily Workflow — Quick Reference

Run every morning between 7:00 – 10:00 AM ET · Full SOP: docs/VA_DAILY_WORKFLOW.md

1. Daily Steps at a Glance

  1. Remote into Ryan's Mac (Tailscale / Chrome Remote Desktop)
  2. Verify 6 AM cron ran — run Command A below
  3. Confirm today's list is in DataSift (app.reisift.io → Records → "H3 Montgomery Courthouse Data")
  4. Wait 30–45 min for DataSift Skip Trace (Activity tab shows "Complete")
  5. Export enriched CSV from DataSift (Manage → Export → download)
  6. Run Command C below with the CSV path
  7. Rename output to Montgomery_Dial_List_YYYY-MM-DD.csv and deliver to dial team

2. The Three Commands (paste-ready)

Command A — Verify the 6 AM cron produced today's file
ls -lt /Users/ryanhawker/SiftStack/output/OH_Montgomery_daily_*.csv | head -1
The date on the top file should be today. If not, run Command B.
Command B — Manual Pass 1 (only if 6 AM cron didn't run; takes 30–40 min)
bash /Users/ryanhawker/SiftStack/scripts/run_ohio_daily_via_chrome.sh --two-pass
Command C — Pass 2 (after DataSift Skip Trace completes; replace PATH)
cd /Users/ryanhawker/SiftStack && TRESTLE_ENABLED=1 PYTHONPATH=src .venv/bin/python -u src/ohio_pass2.py --csv "PATH_TO_DATASIFT_EXPORT.csv"
Get the CSV path: Right-click the downloaded file in Finder → hold Option → "Copy as Pathname" → paste between the quotes.

5. Success Check

  • Pass 2 output shows datasift_hits > 0
  • Final CSV has Owner First/Last populated on most rows

  • Total records matches DataSift list count

6. Ping Ryan Immediately If…

  • 6 AM cron missed AND Command B also fails
  • DataSift Skip Trace stuck "In Progress" for > 2 hours
  • Command C errors twice with the same message
  • Any single-day Pass 2 cost > $5 (normal is $1–$2)
  • Chrome won't launch or crashes repeatedly
  • "Tracerfy 401" or "Trestle 402" errors → API tokens expired
